El problema de la Recomendacin

El escenario que ustedes deberan enfrentar es aquel en que dado un usuario, su historia de preferencias y la informacin de otros usuarios, se requiere entregarle buenas sugerencias. Estas sugerencias deben ser coherentes con los gustos del usuario y para ello deberan ser obtenidas algn tipo de criterio. Por ejemplo, este criterio puede considerar aquellos tems que maximizacen de la valoracin esperada por parte del usuario o que tengan informacin textual (tags por ejemplo) que se aleje minimamente de aquella asociada al usuario.

Ejemplos de este problema pueden observarse en muchos dominios distintos. Por ejemplo, recomendacin de productos a usuarios en sitios de ventas en lnea (amazon, ebay, falabella on-line ), recomendacin de contenido en plataformas multimedia que consiga mantener a los usuarios consumiendo. El insumo (input) para este tipo de mtodos consiste de informacin relativa a los usuarios, tems, contextos de consumo y retroalimentacin de los usuarios luego de interactuar con los tems.

Evaluacin de recomendaciones

Consideremos la matriz sparse presentada antes.

Para evaluar la calidad de las recomendaciones realizadas por cualquier mtodo que construyamos utilizaremos una estrategia _de validacin cruzada Hold-out. Para implementar esta estrategia deberemos dividir (al azar) los datos (las valoraciones) en \(3\) partes: Entrenamiento (\(\%80\)), Prueba (\(\%10\)) y Validacin (\(\%10\)).

set.seed(1)

val_index <- createDataPartition(y = movielens$rating, 
                                  times = 1, p = 0.1, list = FALSE)
tr_tst_set <- movielens[-val_index,]
temp <- movielens[val_index,]

# Nos aseguramos de que cada usuario y cada pelcula esten en ambos segmentos
validation <- temp %>% 
  semi_join(tr_tst_set, by = "movieId") %>%
  semi_join(tr_tst_set, by = "userId")

# Agregamos lo que fue filtrado del segmento de validacion al set de entr. y prueba
removed <- anti_join(temp, validation)
## Joining, by = c("userId", "movieId", "rating", "timestamp", "title", "genres")
tr_tst_set <- rbind(tr_tst_set, removed)

Para continuar con la generacin del conjunto de entrenamiento y prueba, deberemos dividir tr_tst_set en dos partes (entrenamiento y prueba).

# Entrenamiento 90% y prueba 10% (a partir del segmento que contena el 90% de los datos).
set.seed(1)
test_index <- createDataPartition(y = tr_tst_set$rating, times = 1, p = 0.1, list = FALSE)
train_set <- tr_tst_set[-test_index,]
temp <- tr_tst_set[test_index,]

# Make sure userId and movieId in test set are also in train set
test_set <- temp %>% 
  semi_join(train_set, by = "movieId") %>%
  semi_join(train_set, by = "userId")

# Add rows removed from test set back into train set
removed <- anti_join(temp, test_set)
## Joining, by = c("userId", "movieId", "rating", "timestamp", "title", "genres")
train_set <- rbind(train_set, removed)


rm(val_index, test_index, tr_tst_set, temp, removed)

Entrenando y midiendo la calidad de los modelos

Primero, la calidad de las recomendaciones ser medida mediante el Error absoluto promedio y la raz del error cuadrtico medio.

MAE <- function(true_ratings, predicted_ratings){
  mean(abs(true_ratings - predicted_ratings))
}

MSE <- function(true_ratings, predicted_ratings){
  mean((true_ratings - predicted_ratings)^2)
}

RMSE <- function(true_ratings, predicted_ratings){
  sqrt(mean((true_ratings - predicted_ratings)^2))
}

Entonces, para construir cualquier modelo se usar el segmento de entrenamiento. Luego, para seleccionar aquella configuracin de parmetros que permite obtener el mejor modelo se usar el segmento de prueba.

Finalmente, para evaluar la capacidad de generalizacin de cada modelo seleccionado se usar el segmento de validacin.
